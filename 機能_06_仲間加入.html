機能_06_仲間加入

1. シナリオ

  • ターゲット: 探索中のすべてのプレイヤー
  • 状況: 日本全国にはこの勇者PTに限らず無数の人々が石化しており、オーバーワールドや詳細マップには自由に行動しているキャラクターも点在している
  • 使い方: 石化キャラクターを救出するか、歩いている・滞在しているキャラクターに出会うことで仲間に加入する

2. 概要

仲間はすべてのスポット・マップに点在しており、プレイヤーが探索するほど出会いの機会が増える。
加入ルートは「石化救出型」と「フィールド遭遇型」の2種類があり、それぞれ異なるアプローチで仲間を集める。
加入した仲間は 機能_04_派遣・救助.md の派遣機能で使用可能になる。

3. 詳細仕様

3.1 仲間の種類

種別 説明 加入ルート
石化キャラ（ストーリー仲間） 全国の石化キャラの中でもメインクエスト対象の縁の深い仲間3人。詳細は ゲームフロー_02_ストーリークエスト.md 参照 石化救出型
石化キャラ（一般） 地方ごとに1〜2名配置された石化したキャラクター。ストーリー仲間3人もこの枠に含まれる 石化救出型
フィールドキャラ オーバーワールド・詳細マップを歩いている/滞在しているキャラクター フィールド遭遇型

3.2 石化救出型

世界観

チュートリアルのボス戦以前から、日本全国には魔物によって石化させられた無数の人々が点在している。
プレイヤーが各地を探索すると、いたるところで石化したキャラクターのシンボルに出会う。

石化キャラクターの配置

  • 配置場所: 各地方内の任意のスポット（駅・神殿・公園・路上・詳細マップ内など）
  • 配置単位: 日本の地方区分（8地方）ごとに2〜3名を配置。全国で約24名
  • 配置方法: 全プレイヤー共通の固定配置（ランダムではない）
  • 表示: オーバーワールド・詳細マップ上に石化シンボルとして表示される
地方 配置数
北海道 2〜3名
東北 2〜3名
関東 2〜3名
中部 2〜3名
近畿 2〜3名
中国 2〜3名
四国 2〜3名
九州・沖縄 2〜3名

救出フロー

石化シンボルに近づく（半径Xm以内）
    ↓
救出アクションが可能になる
    ↓
【条件分岐】
  YES：救出条件を満たしている → 救出イベント → 仲間加入
  NO ：救出条件を満たしていない → 条件が表示される

救出条件

石化を解くには専用アイテムの所持が必須条件となる。アイテムなしでは救出不可。

項目 内容
必須条件 「石化解除薬」を所持していること
アイテム消費 救出時に石化解除薬を1個消費する
アイテム種別 すべての石化キャラクターに共通の1種類
  • 石化解除アイテムはフィールド探索・ボス撃破・お使いクエストなどで入手できる
  • アイテムを持っていない状態でシンボルに近づいた場合は「◯◯が必要だ」と表示して救出不可を通知する

3.3 フィールド遭遇型

世界観

石化を免れた人々や旅人が、オーバーワールドや詳細マップを自由に行動している。
彼らに話しかけると、仲間として加入してくれることがある。

フィールドキャラクターの種類

出現場所 キャラクタータイプ 特徴
オーバーワールド 旅人・冒険者 マップ上を移動している。近づいて話しかける
詳細マップ（酒場・商店） 滞在キャラ 建物内に留まっている。詳細マップに入ると発見できる
詳細マップ（野外） 探索キャラ 詳細マップ内を歩き回っている

遭遇・加入フロー

オーバーワールド移動中 / 詳細マップ探索中
    ↓
フィールドキャラクターのシンボルを発見
    ↓
話しかけるアクションを実行
    ↓
会話イベント（キャラクター固有の台詞）
    ↓
【加入ルート分岐（キャラクターによって異なる）】
  ├─ 会話型     → 会話を最後まで進めると加入
  ├─ 戦闘型     → キャラクターとバトル → 勝利すると加入
  └─ お使い型   → キャラクターの依頼をクリア → 報告すると加入

加入条件タイプ詳細

タイプ 内容 備考
会話型 話しかけて会話を完了すると加入 最もシンプル。序盤キャラに多い
戦闘型 キャラクターとバトルして勝利すると加入 腕試し系。強いキャラに多い
お使いクエスト型 キャラクターから依頼を受けてクリアすると加入 アイテム収集・場所訪問・モンスター討伐など
  • 1キャラクターにつき加入タイプは1種類
  • お使いクエストの内容はキャラクターごとに固有のものを設定する

3.4 加入共通処理

どちらのルートで加入しても、以下の処理が実行される。

  1. 加入演出（カットシーン or 台詞）
  2. 「〇〇がパーティに加わった！」通知
  3. 仲間一覧に追加
  4. 派遣機能で使用可能になる

4. マスターデータ定義

4.1 companion_master（マスターデータ）

カラム項目 型 取り得る値 / 説明
companion_id int 仲間ID
name string キャラクター名
join_type enum petrified（石化救出型）/ field（フィールド遭遇型）
spawn_spot_id int 出現スポットID（石化型は石化場所、フィールド型は出現エリア）
join_type_detail enum talk（会話型）/ battle（戦闘型）/ errand（お使い型）
rescue_item_id int 固定値：石化解除薬のアイテムID
is_story bool ストーリー仲間（メインクエスト対象）フラグ

4.2 user_companion（ユーザーデータ）

カラム項目 型 取り得る値 / 説明
user_id int プレイヤーID
companion_id int 仲間ID
status enum unknown（未発見）/ found（発見済・未加入）/ joined（加入済）
joined_at datetime 加入日時

5. 画面遷移フロー

オーバーワールド / 詳細マップ探索
    ↓
石化シンボル or フィールドキャラクターを発見
    ↓
近づく / 話しかける
    ↓
加入条件を満たしている？
  ├─ YES → 加入イベント → 仲間一覧に追加
  └─ NO  → 条件テキスト表示 → マップに戻る

- ポップアップ外タップ - - ポップアップを閉じる

6.2 加入通知

No. パーツ名 種別 内容 処理
1 加入テキスト ラベル 「〇〇がパーティに加わった！」 数秒後に自動消去
2 キャラクターアイコン 画像 加入したキャラクターのアイコン -

7. 対象外（Out of Scope）

  • 仲間のパラメータ・スキル詳細（params_character.gsheet で管理）
  • 仲間の離脱・解雇機能（未検討）
  • 仲間との好感度・信頼度システムの詳細（再訪問条件の実装に必要なため別途定義）
  • 仲間一覧画面の詳細レイアウト（別仕様書で定義予定）

8. 未解決の問題

  • 石化解除薬の入手方法・入手頻度のバランスが未定
  • 戦闘型加入キャラクターのバトル仕様（負けたらどうなるか、再挑戦できるか）が未定
  • お使いクエストの管理をストーリークエストと同じ仕組みにするか別管理にするか未定
  • 石化キャラの総数は全国で約24名（ストーリー仲間3名を含む）
  • 各地方の配置数（2名か3名か）と具体的なスポットが未定
  • ストーリー仲間3名をどの地方に配置するか未定
  • フィールドキャラクターのリスポーン・再出現ルールが未定（一度発見したら固定か、移動するか）
  • 再訪問による信頼度制の詳細仕様が未定
  • 仲間の上限人数が未定
  • 石化キャラとフィールドキャラの比率・バランスが未定

9. 用語解説

用語 説明
石化救出型 石化したキャラクターに近づいて救出することで仲間にするルート
フィールド遭遇型 オーバーワールドや詳細マップを自由に行動しているキャラクターに話しかけて仲間にするルート
ストーリー仲間 メインクエストで救出する縁の深い仲間3人。詳細は ゲームフロー_02_ストーリークエスト.md を参照
